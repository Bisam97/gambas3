I want to open a file with the OPEN instruction, and I get this error message : "Type mismatch. Wanted File, got Integer instead"

The OPEN instruction of Gambas does not work like the Visual Basic one. It does not return the file as an integer, but as a File object.

Instead of typing :

  DIM handle AS Integer
  ...
  OPEN "myfile" FOR READ AS #handle

you must type :

  DIM handle AS File
  ...
  OPEN "myfile" FOR READ AS #handle

I have defined a public variable in a module, but when I want to use it from another module, I get an "undefined identifier" error.

Gambas works more like Java than like Visual Basic.

I mean that a module is exactly like a class, except that all its members are static.

So, if you declare a variable in a module named MyModule this way :

  PUBLIC MyVariable AS String

then to access it, you must use the following syntax :

  PRINT MyModule.MyVariable

There are no global public variables like in Visual Basic, because I think it is a completely stupid and useless feature. Trust me ! ;-)
