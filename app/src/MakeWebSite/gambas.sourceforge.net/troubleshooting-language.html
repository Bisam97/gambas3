I built a simple Gambas application by scrupulously following one of the examples. The application compiles, but don't run. What's wrong ?

You must define a startup class in the project properties dialog, and this startup class must have a static public method named Main without any argument to act as a startup method.

A bug in the development environment prevents it from displaying a warning message to the user. It is the reason why nothing happens when you run the project.

I created a small project with only one form, and I define this form as the startup class. When I run the project, nothing happens ! Why ?

Contrary to Visual Basic, Gambas form are not self-executable. As for any other classes, you must define a static public method named Main so that it works. See above for more details.

I want to open a file with the OPEN instruction, and I get this error message : "Type mismatch. Wanted File, got Integer instead"

The OPEN instruction of Gambas does not work like the Visual Basic one. It does not return the file as an integer, but as a File object.

Instead of typing :

  DIM handle AS Integer
  ...
  OPEN "myfile" FOR READ AS #handle

you must type :

  DIM handle AS File
  ...
  OPEN "myfile" FOR READ AS #handle

When I compile my project, I got an "undefined identifier" error on a well known class name that seems absolutely correct.

For example, with the following code :

  MyControl.Font = font["Luxi Sans 16"]

When the compiler encounters an identifier, it assumes reading a class name if the identifier begins with a capital letter.

You have two solutions to solve this problem.

First you can change the identifier first letter, like that :

  MyControl.Font = Font["Luxi Sans 16"]

Secondly, you can predeclare the identifier as a class, with the CLASS keyword. This declaration must be placed at the beginning of the class.

  CLASS font
  ...
  MyControl.Font = font["Luxi Sans 16"]
